# 변수명 VS 메소드명
* 변수는  `명사`를 주로 사용
* 메소드는 `동사`를 주로 사용
  
# 메서드 사용의 장점
* **코드의 재사용성** : 메서드를 **캡슐화**하므로 필요할때 마다 호출 가능
* **코드의 가독성**: 코드가 수행되는 영역을 정확하게 알아 **유지보수가 쉬움**
* **모듈성**: 큰 프로그램에서 역할을 나눌수있음,**가독성과 디버기잉 쉬움**
* **코드 유지 관리**:전체코드의 영향을 주지 않고 **특정 메서드만 수정가능**
* **재사용성과 확장성**:다른 잘 설계된 메서드를 **공유,재사용 가능,메서드 확장 가능**
* **추상화**:메서드의 내용을 **정확히 몰라도 사용가능**
* **테스트와디버깅 용이성**:개발 메소드는 독립적으로 디버깅 및 테스트 할수있어 **신속하게 수정 가능하다**
